## Prerequisites

- **Node.js** v20+ (v24 recommended)
- **GitHub CLI** (`gh`) authenticated with Copilot access
- **Git** for worktree management

## Installation

```bash
# Clone the repository
git clone https://github.com/michaeljolley/io.git
cd io

# Install dependencies
npm install

# Build all packages
npm run build
```

## Running the Daemon

```bash
# Start in development mode (auto-reload)
npm run dev

# Or start the built daemon
node packages/daemon/dist/index.js
```

The daemon starts on port **7777** by default.

## Open the Web Dashboard

Once the daemon is running, open your browser to:

```
http://localhost:7777
```

The web dashboard provides a full-featured UI for chatting with IO, managing squads, viewing the inbox, and more.

If you've configured Supabase (see [Configuration](/io/guides/configuration/)), you'll be prompted to log in. Without Supabase, the dashboard is open.

## Verify It's Running

```bash
curl http://localhost:7777/api/health
```

You should see:
```json
{
  "status": "healthy",
  "uptime": 5,
  "copilotConnected": true
}
```

## Connect with the TUI

```bash
cd packages/tui
npm start
```

This opens a full-screen terminal interface where you can chat with IO.

## Configuration

IO stores its data in `~/.io/` by default. Override with the `IO_DATA_DIR` environment variable.

| Environment Variable | Default | Description |
|---------------------|---------|-------------|
| `IO_API_PORT` | `7777` | API server port |
| `IO_LOG_LEVEL` | `info` | Log level (debug, info, warn, error) |
| `IO_DATA_DIR` | `~/.io` | Data directory path |
